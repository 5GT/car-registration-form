# 직원용 차량등록 운영 안내서

## 1. 이 시스템의 목적
이 시스템은 입주민이 차량등록 신청서를 미리 작성하도록 해서 관리사무소 방문 시 혼잡을 줄이고, 직원이 신청 내역을 빠르게 조회하여 서류 확인 후 차량등록카드(스티커)를 발급할 수 있도록 돕기 위한 용도입니다.

핵심 흐름은 아래와 같습니다.
1. 입주민이 사전에 신청서를 작성합니다.
2. 신청 내용이 데이터베이스에 저장됩니다.
3. 입주민이 관리사무소를 방문합니다.
4. 직원이 신청 내용을 조회합니다.
5. 직원이 주소 및 구비서류를 확인합니다.
6. 확인이 끝나면 차량등록카드(스티커)를 발급합니다.

## 2. 사용 대상
- 입주민: 신청서 작성
- 관리사무소 직원: 신청 내용 조회, 확인, 수정, 삭제, 출력

## 3. 페이지 구성
- `index.html`: 메인 진입 페이지
- `resident.html`: 입주민 차량등록 신청 페이지
- `external.html`: 외부 차량 등록 페이지
- `complete.html`: 신청 완료 안내 페이지

## 4. 입주민 차량등록 절차
입주민은 아래 순서로 신청합니다.
1. 신청 페이지에 접속합니다.
2. 동과 호를 선택합니다.
3. 차량 정보를 입력합니다.
4. 소유자, 사용자, 연락처, 보유형태를 입력합니다.
5. 차량구분을 선택합니다.
6. 필요한 경우 메모를 입력합니다.
7. 신청서를 제출합니다.
8. 완료 화면에서 접수 내용을 확인합니다.

## 5. 직원 확인 절차
직원은 아래 순서로 업무를 진행합니다.
1. 관리자 페이지에서 신청 내역을 조회합니다.
2. 입주민의 동, 호, 차량번호, 신청인 정보를 확인합니다.
3. 방문한 입주민이 제출한 구비서류를 확인합니다.
4. 차량 소유자 또는 사용자의 주소가 해당 아파트 기준에 맞는지 확인합니다.
5. 필요 시 신청 내용을 수정합니다.
6. 서류 이상이 없으면 차량등록카드(스티커)를 발급합니다.
7. 필요 시 처리 상태나 메모를 남깁니다.

## 6. 직원이 확인해야 할 기본 항목
- 동 / 호
- 차량번호
- 차명
- 소유자 성함
- 실제 사용자 성함
- 연락처
- 보유형태
- 차량구분
- 제출서류 적합 여부
- 주소 확인 여부

## 7. 차량 등록 기준 메모
- 한 세대당 차량은 최대 3대까지 등록 가능합니다.
- 이미 등록된 차량번호는 중복 등록되지 않도록 확인해야 합니다.
- 입주민 페이지는 공개용이므로 조회, 수정, 삭제 기능이 없습니다.
- 관리자 페이지에서만 조회, 수정, 삭제가 가능합니다.

## 8. 테스트 및 확인 방법
### 입주민 기능 확인
- 시크릿 모드에서 테스트합니다.
- 입주민 신청 페이지에서 실제 제출이 되는지 확인합니다.
- 같은 세대 기준으로 4대째 등록 시 제한 메시지가 나오는지 확인합니다.

### 관리자 기능 확인
- 일반 브라우저에서 관리자 계정으로 로그인합니다.
- 신청 내역 조회가 되는지 확인합니다.
- 수정 및 삭제가 가능한지 확인합니다.

## 9. 주의사항
- 입주민 기능과 관리자 기능은 같은 브라우저 세션에서 섞어 테스트하지 않는 것이 좋습니다.
- 관리자 로그인 상태에서 입주민 페이지를 시험하면 결과가 혼동될 수 있습니다.
- 공개 페이지 코드에는 `service_role` 키를 넣으면 안 됩니다.
- 권한 정책과 트리거 변경이 필요할 경우 SQL 파일 기준으로 관리합니다.

## 10. 저장소 안의 참고 파일
- `supabase_rls_plan.sql`: 권한 분리 참고
- `car_registrations_public_patch.sql`: 공개 등록 관련 SQL
- `car_registrations_household_limit_patch.sql`: 세대당 3대 제한 SQL
- `car_registrations_compact_discount_patch.sql`: 경차 할인 컬럼 SQL
- `car_registrations_household_notice_patch.sql`: 세대알림 컬럼 SQL
- `external_car_registrations.sql`: 외부 차량 등록 SQL
